DEPLOYMENT PROCESS (Step-by-Step):
================================================================================

STEP 1: GitHub Setup (5 minutes)
───────────────────────────────
Commands to run locally:

git init
git add .
git commit -m "Initial commit - SailAnime anime streaming platform"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/sailanime.git
git push -u origin main

Note: Replace YOUR_USERNAME with your GitHub username


STEP 2: Create Render Web Service (10 minutes)
──────────────────────────────────────────────
1. Visit https://render.com
2. Click "New +"
3. Select "Web Service"
4. Connect GitHub (authorize if needed)
5. Select "sailanime" repository
6. Configure:
   - Name: sailanime (or preferred name)
   - Environment: Python 3
   - Build Command: pip install -r requirements.txt
   - Start Command: python app.py
   - Region: Singapore (closest to Uzbekistan)
   - Plan: Free (or Paid for production)
7. Click Deploy


STEP 3: Add Environment Variables (5 minutes)
──────────────────────────────────────────────
In Render dashboard, go to Web Service Settings > Environment:

1. DATABASE_URL: postgresql://[connection_string]
   - Render will provide this if using Render PostgreSQL
   - Or provide external database connection string

2. FLASK_ENV: production

3. PORT: 5000 (usually auto-configured)

Click Deploy after changes.


STEP 4: Create PostgreSQL Database (5 minutes - if needed)
──────────────────────────────────────────────────────────
Option A: Render PostgreSQL (Recommended)
- New PostgreSQL service in Render
- Connection string auto-linked

Option B: External Database
- Use existing PostgreSQL
- Provide CONNECTION STRING in DATABASE_URL


STEP 5: Test Deployment (10 minutes)
─────────────────────────────────────
1. Wait for deployment to complete (check logs)
2. Open your deployed URL
3. Test features:
   ✓ Homepage loads
   ✓ Cards display
   ✓ Hero slider works
   ✓ Search works
   ✓ Admin login (password: admin123)
   ✓ Add video link
   ✓ Mobile responsive
   ✓ No console errors


STEP 6: Update Sitemap & SEO (10 minutes)
──────────────────────────────────────────
1. Copy your deployed domain (e.g., sailanime-abc123.onrender.com)
2. Update sitemap.xml with actual domain
3. Push to GitHub
4. Render auto-re-deploys
5. Submit to Google Search Console

TROUBLESHOOTING GUIDE:
================================================================================

Common Issues:

1. Deployment fails
   - Check logs in Render dashboard
   - Verify procfile syntax
   - Ensure requirements.txt is valid
   - Check for typos in commands

2. Database connection error
   - Verify DATABASE_URL environment variable
   - Check PostgreSQL service status
   - Ensure credentials are correct
   - Test connection string locally

3. Static files not loading (CSS/JS)
   - Check file names (case-sensitive)
   - Verify app.py static folder configuration
   - Check browser cache (Ctrl+Shift+Delete)
   - Look at network tab in DevTools

4. Admin panel won't load
   - Check browser console for JS errors
   - Verify password is "admin123"
   - Clear browser cache
   - Test in incognito window

5. Slow loading
   - Check Render analytics
   - Optimize image sizes
   - Use CDN for assets
   - Consider Pro plan
